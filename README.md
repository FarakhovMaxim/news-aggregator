# 📰 News Aggregator

Консольное Java-приложение для автоматического сбора, хранения, поиска и анализа новостей из RSS-лент Lenta.ru и RBC.

## ✨ Возможности

| Категория | Что умеет |
|---|---|
| **Сбор данных** | Парсинг RSS Lenta.ru и RBC через Jsoup, многопоточная загрузка (ExecutorService) |
| **Хранение** | SQLite (файл `news.db`), дедупликация по `link`, таблицы `articles` + `images` |
| **Поиск** | По ключевым словам, категории, источнику, дате |
| **Сортировка** | По дате, источнику, популярности (длина текста) |
| **Аналитика** | Статистика по категориям, топ слов (стоп-слова отфильтрованы), динамика упоминаний по дням |
| **Экспорт** | JSON (Gson) и CSV |
| **Логирование** | Добавление и удаление новостей пишутся в `news_history.log` |
| **Автообновление** | Фоновое обновление по расписанию через `ScheduledExecutorService` |
| **Очистка** | Удаление новостей старше N дней |

## 🏗 Архитектура

```
com.newsaggregator/
│
├── app/
│   └── Main.java                  ← точка входа
│
├── ui/
│   └── ConsoleApp.java            ← консольное меню, ввод/вывод
│
├── service/
│   ├── NewsService.java           ← оркестрация: загрузка + сохранение
│   ├── CleanupService.java        ← удаление устаревших новостей
│   ├── ScheduledUpdater.java      ← автообновление по расписанию
│   └── ChangeLogger.java          ← запись событий в news_history.log
│
├── db/
│   ├── ArticleDb.java             ← CRUD-операции, инициализация схемы
│   └── Filter.java                ← параметры поиска и сортировки
│
├── parser/
│   ├── RssParser.java             ← интерфейс парсера
│   ├── LentaParser.java           ← парсер Lenta.ru RSS
│   └── RbcParser.java             ← парсер RBC RSS
│
├── analyzer/
│   └── StatisticsAnalyzer.java    ← аналитика: категории, слова, динамика
│
├── export/
│   └── NewsExporter.java          ← экспорт в JSON и CSV
│
├── http/
│   └── HttpFetcher.java           ← HTTP-клиент (java.net.http)
│
└── model/
    └── Article.java               ← сущность новости
```

**Поток данных:**

```
RSS URL
  └─► HttpFetcher.fetch()
        └─► LentaParser / RbcParser  (Jsoup XML)
              └─► NewsService.save()
                    └─► ArticleDb  (SQLite)
                          ├─► StatisticsAnalyzer
                          ├─► NewsExporter (JSON / CSV)
                          └─► ConsoleApp (вывод в терминал)
```

## 🗄 База данных

Файл `news.db` создаётся автоматически при первом запуске.

### Таблица `articles`

| Колонка | Тип | Описание |
|---|---|---|
| `id` | INTEGER PK | Автоинкремент |
| `title` | TEXT | Заголовок |
| `description` | TEXT | Краткое описание |
| `full_text` | TEXT | Полный текст |
| `link` | TEXT UNIQUE | URL статьи (ключ дедупликации) |
| `date` | TEXT | Дата публикации (ISO: `yyyy-MM-dd`) |
| `category` | TEXT | Категория |
| `source` | TEXT | Источник (`Lenta.ru` / `RBC`) |

### Таблица `images`

| Колонка | Тип | Описание |
|---|---|---|
| `id` | INTEGER PK | Автоинкремент |
| `article_id` | INTEGER FK | → `articles.id` |
| `url` | TEXT | URL изображения |

## ⚙️ Требования и установка

### Требования

- **Java 17+**
- **Maven 3.8+**
- Интернет-соединение (при первом запуске для загрузки новостей)

## 🚀 Запуск

### Сборка fat-JAR

```bash
mvn clean package -q
```

Собранный JAR с зависимостями: `target/news-aggregator-jar-with-dependencies.jar`

### Запуск

```bash
java -jar target/news-aggregator-jar-with-dependencies.jar
```

После запуска автоматически:
1. Создаётся (или открывается) `news.db`
2. Запускается фоновый планировщик обновлений
3. Открывается консольное меню

## 🧪 Тесты

Тесты написаны на **JUnit 5 + Mockito** и покрывают все основные слои:

| Тест-класс | Что проверяется |
|---|---|
| `LentaParserTest` | Парсинг XML: заголовок, дата, категория, изображения, дубликаты, пустой XML |
| `RbcParserTest` | Парсинг XML RBC: те же сценарии + специфика формата |
| `FilterTest` | Фильтрация по ключевым словам, категории, источнику, дате; сортировка по всем трём осям |
| `StatisticsAnalyzerTest` | Подсчёт по категориям, топ слов, динамика упоминаний, фильтрация стоп-слов |
| `NewsExporterTest` | Экспорт в JSON (структура, поля, массив), экспорт в CSV (заголовок, строки, экранирование запятых) |
| `ChangeLoggerTest` | Запись события добавления, запись удаления, формат строки лога, создание файла |

### Запуск тестов

```bash
# Все тесты
mvn test

# Конкретный класс
mvn test -Dtest=LentaParserTest

# С подробным выводом
mvn test -Dsurefire.useFile=false
```
